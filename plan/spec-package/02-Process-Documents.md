# 过程驱动文档（Process Documents）

## 1. 执行节奏（建议5-7天）
- D1：需求冻结 + 原型确认 + 技术选型确认。
- D2-D3：上传页开发与联调（含限制规则）。
- D3-D4：识别编辑页开发（左图右文、状态机、重试）。
- D5：导出模块（docx/pdf）与端到端联调。
- D6-D7：测试修复、埋点校验、发布准备。

## 2. 研发任务拆解（可直接给 Agent）

## 2.1 前端任务
- T1 上传模块
  - 实现上传组件（点击/拖拽/多选）。
  - 实现图片列表和缩略图预览。
  - 实现 `isPro` + 10张限制逻辑。
  - 实现格式/大小校验与错误提示。
- T2 识别模块
  - 封装 `runOcr(file)` 服务层。
  - 实现单图识别状态管理（processing/success/error）。
  - 实现批量串行或并行识别策略（MVP建议并发2）。
- T3 编辑模块
  - 实现左图右文卡片。
  - 文本编辑实时更新状态（防抖500ms持久化）。
  - 失败页重试按钮和结果覆盖。
- T4 导出模块
  - 实现 `buildDocx(items)`。
  - 实现 `buildPdf(items)`。
  - 导出前检查空页并弹窗提醒。
- T5 埋点与异常
  - 埋点：上传数、识别耗时、导出类型、导出结果。
  - 全局错误边界和OCR失败日志上报。

## 2.2 后端预留任务（可后置）
- B1 用户计划接口：返回 `isPro`。
- B2 OCR批量接口：返回按上传顺序的文本结果。
- B3 鉴权校验：服务端再次校验上传上限（防前端绕过）。

## 3. 角色分工
- PM：需求冻结、验收标准定义、上线范围控制。
- FE：页面开发、状态管理、导出实现、埋点接入。
- QA：流程回归、边界条件、导出文件校验。

## 4. 风险与应对
- OCR质量不稳定：提供人工编辑优先级，支持重试。
- 浏览器内存压力：限制单图大小与并发识别数。
- 会员逻辑被绕过：生产阶段必须接后端校验。
- PDF中文字体问题：内置中文字体或服务端导出兜底。

## 5. 测试清单（最小集）
- 上传 1/10/11 张边界测试。
- 混合合法与非法文件上传测试。
- 多张识别中断后恢复与重试测试。
- 编辑后导出内容一致性测试。
- `docx/pdf` 在常见阅读器可打开测试。
